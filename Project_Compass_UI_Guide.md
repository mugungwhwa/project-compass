## 1. 핵심 원칙: Analytics Dashboard가 아니라 Decision Dashboard

```
Analytics Dashboard               Decision Dashboard (Compass)
━━━━━━━━━━━━━━━━━━               ━━━━━━━━━━━━━━━━━━━━━━━━━━━
"무슨 일이 일어났는가?"            "어떻게 해야 하는가?"
데이터 → 사용자가 해석              시스템이 해석 → 추천 제공
차트 우선                           시그널 우선
탐색형 (Explore)                    확인형 (Confirm)
인지 부하: 높음                     인지 부하: 낮음
```

**모든 화면은 "Should we...?" 질문으로 시작해야 합니다.**
- Module 1: "현재 투자 포지션은 어떤가?" (요약 레이어)
- Module 2: "시장 대비 어디로 향하고 있는가?"
- Module 3: "어떤 액션이 실제로 가치를 만들고 있는가?" (가치 창출 레이어)
- Module 4: "실험이 투자 가능성을 높이고 있는가?" (가치 창출 레이어)
- Module 5: "다음 예산은 어디로 옮겨야 하는가?"

### 2.2 왜 우측 사이드바인가

| 근거 | 설명 |
|---|---|
| **읽기 흐름** | LTR 언어에서 눈은 좌측에서 시작 → 가장 중요한 콘텐츠(결과/시각화)가 먼저 보임 |
| **Decision-first** | 좌측 = 결과(투자 판단), 우측 = 조건(필터/설정) → "결론 먼저, 조건 나중에" |
| **트레이딩 메타포** | Bloomberg 등 금융 플랫폼에서 우측은 "주문/액션 패널" → 투자 결정 도구로 자연스러운 메타포 |
| **차별화** | 대부분의 SaaS가 좌측 사이드바 → Compass는 시각적으로 즉시 구별됨 |

### 2.3 사이드바 설계 상세

```
Width:     300px (고정), 48px (접힌 상태, 아이콘만)
Toggle:    Cmd+\ 또는 ] 키 (Linear 컨벤션)
Content:   모듈에 따라 Context 영역이 동적으로 변경

항상 표시:
  - 모듈 네비게이션 (5개)
  - 알림 벨 (실험 결과, 벤치마크 이탈, 이상치)
  - 사용자/팀 설정

모듈별 Context 변경:
  Overview:    게임 선택, 날짜 범위, 신뢰 수준 토글
  Market Gap:  벤치마크 토글, 경쟁사 선택, 장르 필터
  Actions:     액션 유형 필터, 기간 선택
  Experiments: 실험 상태 필터, 약식 실험 추가 버튼
  Capital:     시나리오 파라미터 슬라이더, 리스크 허용도
```

### 2.4 반응형 전략

| 화면 | 사이드바 | 콘텐츠 |
|---|---|---|
| **Desktop (>1280px)** | 전체 표시 (300px) | 나머지 영역 |
| **Tablet (768-1280px)** | 접힌 상태 (48px), 클릭 시 오버레이 | 전체 너비 |
| **Mobile (<768px)** | 하단 네비게이션 바 (5개 모듈) | 전체 너비, 필터는 상단 시트 |

## 3. 투자 시그널 시각화

### 3.1 트래픽 라이트를 쓰지 않는 이유

단순 빨/노/초 원형은 과도한 단순화. 임계값 경계에서 정보를 잃음. 신뢰도를 표현하지 못함.

### 3.2 권장 패턴: Composite Signal Card

```
┌──────────────────────────────────────────────────────┐
│  INVESTMENT STATUS                                   │
│                                                      │
│  ████████████████████████████  ● Increase rollout    │
│  ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓░░░░░░░░   [GREEN]                │
│  ◄─ Reduce ─┤─ Hold ─┤─ Invest ─►                   │
│              ▲ marker                                │
│                                                      │
│  Decision Confidence: 82%                            │
│                                                      │
│  Key Signals:                                        │
│  ✓ Retention above P75 genre benchmark               │
│  ✓ Payback projected D47 (target: D60)               │
│  ⚠ CPI trending up +8% MoM                          │
│  ✓ Experiment velocity: 3.2 wins/month               │
│                                                      │
│  Payback: D47 [D38──────D47──────D62]                │
│                P10       P50      P90                 │
└──────────────────────────────────────────────────────┘
```

### 3.3 시그널 표현 규칙

| 시그널 | 색상 | 아이콘 | 텍스트 예시 | 배경 |
|---|---|---|---|---|
| **GREEN** | `#16A34A` | ✓ 체크마크 | "Increase rollout" / "Scale UA" | `#DCFCE7` |
| **YELLOW** | `#D97706` | ⚠ 삼각형 | "Scale with caution" / "Keep testing before scaling" | `#FEF3C7` |
| **RED** | `#DC2626` | ✕ 정지 | "Pause scaling" / "Reduce exposure" | `#FEE2E2` |

**시그널 텍스트 원칙**: 단독 투자 판단("INVEST MORE")보다 구체적 다음 액션("Increase rollout", "Keep testing before scaling")을 우선 표기. 시그널은 Executive Overview의 요약이며, 실제 가치 창출 근거는 Module 3(Actions)과 Module 4(Experiments)에서 도출됨.

**색맹 접근성**: 색상만으로 구분하지 않음. 반드시 아이콘 + 텍스트 + 위치를 함께 사용.
대안 팔레트: Green 대신 Blue(`#5B9AFF`) 사용 → Blue/Amber/Red는 색각 이상에 본질적으로 안전.

## 4. 불확실성 시각화 (P10/P50/P90)

### 4.1 Fan Chart (리텐션/매출 예측)

```
Retention (%)
100│
   │
 50│      ░░░░░░░░░░░░ P90
   │    ░░▓▓▓▓▓▓▓▓▓▓░░
   │  ░░▓▓████████▓▓░░ P50 (실선)
   │░░▓▓████████████▓▓░░
   │▓▓████████████████▓▓ P10
  0│──┬──────────────────── Days
   │  D1  D7  D14  D30    D60   D180
   │
   │  ←── 관측 ──→│←── 예측 ──→
   │  (실선+점)    │  (밴드)
```

### 4.2 밴드 스타일링

```
P50 선:         #5B9AFF (accent-blue, 실선, 2px)   ← deck palette
P25-P75 밴드:   #5B9AFF at 25% opacity
P10-P90 밴드:   #5B9AFF at 12% opacity
벤치마크 밴드:  #94A3B8 at 15% opacity (회색, 게임 데이터와 구별)

관측 데이터:    실선 + 점 (●)
예측 데이터:    "오늘" 기준 수직 점선 이후
```

### 4.3 비통계인을 위한 표현

통계 용어를 쓰지 않습니다:

| 통계 용어 | Compass 표현 |
|---|---|
| P10 / P90 | "보수적 / 낙관적" 또는 "worst realistic / best realistic" |
| 크레디블 인터벌 | "가능한 범위" |
| Posterior | "우리 추정" |
| Prior | "시장 벤치마크" |
| Confidence | "확신도" 또는 퍼센트 바 |

**인라인 범위 표기**:
```
페이백 기간:  47일  (38-62일 범위)
              ^^^^   ^^^^^^^^^^^
              크게/굵게  작게/연하게
```

### 4.4 인터랙션

- **Hover**: 수직 십자선 + 해당 시점의 P10/P50/P90 정확한 값 툴팁
- **밴드 토글**: 헤더에 "범위 표시/숨기기" 토글 → P50 선만 보기 가능
- **시나리오 오버레이**: 시나리오 실행 시 기존 밴드 위에 시나리오 밴드 겹침
- **시간 여행**: "N일 전 예측은 어떠했는가?" 슬라이더 → 베이지안 업데이트 과정 시각화

## 5. Prior vs Posterior 시각화

### 5.1 Compass의 가장 독특한 시각화

베이지안 업데이트의 핵심 — "데이터가 쌓일수록 확신이 커진다"를 눈으로 보여줌.

```
확률밀도
  │
  │         Prior (시장 벤치마크)
  │       ╱  ╲
  │     ╱      ╲            넓고 평평
  │   ╱          ╲
  │ ╱              ╲
  │╱                ╲
  │        ┃
  │      ╱ ┃╲  Posterior (우리 추정)
  │    ╱   ┃  ╲          좁고 높음
  │  ╱     ┃    ╲
  │╱       ┃      ╲
  ├────────┃────────────── D30 Retention (%)
  │   5%   ┃8.3%  12%
  │        ┃
  │     실제 관측값
  │
  │  [───── Prior 90% 범위 ─────]  6.2% - 9.8%
  │     [── Posterior 90% ──]      7.8% - 8.9%
  │
  │  "불확실성 53% 감소"
```

### 5.2 비기술 사용자를 위한 해석 카드

분포 플롯 아래에 자연어 요약:

```
┌──────────────────────────────────────────────────┐
│  📊 분석 요약                                     │
│                                                   │
│  시장 벤치마크(RPG 장르)만으로는 D30 리텐션이      │
│  6.2%-9.8% 범위로 예상되었습니다.                  │
│                                                   │
│  귀사의 실제 D14 코호트 데이터를 반영한 결과,      │
│  7.8%-8.9%로 범위가 좁혀졌습니다.                  │
│                                                   │
│  → 불확실성이 53% 감소했으며,                      │
│    RPG 장르 상위 25% 수준으로 추정됩니다.           │
│                                                   │
│  데이터 기반: 14일 관측, 8개 코호트                │
│  Prior 출처: GameAnalytics 2024 RPG 벤치마크      │
└──────────────────────────────────────────────────┘
```

### 5.3 애니메이션 (선택사항)

첫 로드 시 Prior → Posterior 전환을 애니메이션으로 표현:
넓은 분포가 점차 좁아지면서 "데이터가 확신을 만드는 과정"을 시각적으로 전달.
Framer Motion으로 구현, 0.8초 트랜지션.

## 6. 정보 흐름: Signal → Evidence → Action

### 6.1 최적 정보 순서 (의사결정 연구 기반)

```
1. SIGNAL   → "상황이 어떤가?"         (2초)
2. CONTEXT  → "핵심 수치는?"           (10초)
3. EVIDENCE → "데이터가 뭐라 하는가?"  (30초)
4. OPTIONS  → "무엇을 할 수 있는가?"   (20초)
5. ACTION   → "실행"                   (5초)

총 의사결정 시간: 숙련된 운영자 기준 ~1분
```

### 6.2 Progressive Disclosure (3단계)

| 레벨 | 보이는 것 | 접근 방법 |
|---|---|---|
| **Level 0 — Glance** | 시그널 카드 + 핵심 숫자 1-3개 | 즉시 보임 (스크롤 불필요) |
| **Level 1 — Scan** | KPI 카드 + 차트 요약 + 추천 | 아래로 스크롤 |
| **Level 2 — Analyze** | 전체 리텐션 커브, 코호트 테이블, 실험 상세 | 카드 클릭 "View details" |
| **Level 3 — Investigate** | 원시 데이터, 모델 파라미터, 데이터 신선도 | "Methodology" 탭/패널 |

### 6.3 모바일에서의 정보 우선순위

모바일 화면에서 fold 위에 보여야 하는 것 (우선순위 순):
1. 투자 시그널 (Green/Yellow/Red) + 한줄 요약
2. 오늘 매출 vs 목표
3. 최신 코호트 D1 리텐션 vs 벤치마크
4. 활성 실험 수 + 유의미한 결과 알림

### 6.4 재무 데이터 입력 UX

**원칙: "Value-first, 3분 입력, 범위 허용"**

```
사용자 온보딩 흐름:

Step 1: 게임 등록 (앱 ID 입력 → 자동 메타데이터 수집)
  → 이 시점에서 이미 벤치마크 분석 결과를 보여줌 (Prior only)
  → "귀사 게임은 RPG 장르 상위 30%에 위치합니다"

Step 2: MMP 연동 (AppsFlyer API Key 입력)
  → CPI, ROAS, 코호트 리텐션이 자동으로 채워짐
  → 리텐션 예측 + LTV 계산이 활성화됨

Step 3: 재무 입력 (여기서 처음 요청) ← Value를 먼저 본 후
  → "투자 시그널과 Burn Tolerance를 활성화하려면
     5개 재무 지표를 입력해주세요 (3분 소요)"
```

#### UX 핵심 규칙

| 규칙 | 구현 |
|---|---|
| **Value-first** | MMP 연동 후 벤치마크/리텐션 결과를 먼저 보여주고, 그 후에 재무 요청 |
| **3분 완료** | Tier 1은 5개 필드만 — 복잡한 폼 금지 |
| **범위 허용** | 정확한 수치 대신 min-max 범위 입력 가능 → 심리적 부담 감소 |
| **보안 배지** | 입력 폼 바로 옆에 암호화/감사추적 안내 표시 |
| **삭제 보장** | "언제든 삭제 가능" 명시 → 진입 장벽 제거 |

### 7.4 Module 4: Experiment Investment Board

**핵심 질문**: "R&D가 투자 가치를 만들고 있는가? — 그리고 어떤 variant가 실제로 LTV를 움직였는가?"

#### 7.4.1 상단 레이어 — 포트폴리오 개요
- KPI 카드: Velocity, Ship Rate × Win Rate, 누적 ΔLTV, 평균 Time-to-Decision
- 실험 포트폴리오 테이블 (이름, 상태, ATE, ΔLTV, ROI)
  - 초록 행: shipped winners
  - 회색 행: not shipped
  - 노랑 행: 진행 중
  - 빨강 행: reverted (롤백) — "실수 비용" 컬럼 강조

#### 7.4.2 중단 레이어 — Variant Impact Board (신규)

실험을 클릭하면 우측 슬라이드 패널로 variant 단위 상세 뷰가 열린다.

**Variant Impact Chart (수평 막대 + 에러바)**
- 각 variant를 한 행으로 표시: `Control | V1 | V2 | ...`
- 막대 길이: ΔLTV per user (control 대비)
- 에러바: P10~P90 크레디블 인터벌
- 색상 코드:
  - 회색: control (baseline)
  - 초록: winner (CI 하한 > 0)
  - 빨강: loser (CI 상한 < 0)
  - 점선 초록: shipped (실제 배포됨)
  - 점선 빨강: reverted (배포 후 롤백)
- 각 variant 우측 끝: sample_size, confidence %

**Rollout History Timeline (신규)**
- 가로축: 시간 (shipped_at → 현재 또는 reverted_at)
- 세로축 좌: 롤아웃 비율 (0~100%), 계단식 라인
- 세로축 우: 누적 ΔLTV ($), 영역 차트
- 주요 이벤트 마커: 5% → 15% → 35% → 60% → 100% 확장 지점
- Reverted variant는 롤백 지점에서 빨간 X 마커 + 회색 음영으로 "이후 기간 = 기회비용"

#### 7.4.3 하단 레이어 — Ripple Forecast Fan Chart (신규)

**핵심 차별화**: "이 variant를 지금 50%로 확대하면?"에 대한 답.

- 가로축: 롤아웃 단계 (5% → 25% → 50% → 100%)
- 세로축: 예상 누적 ΔLTV ($)
- Fan 밴드: P10/P50/P90 (기존 Fan Chart 스타일 재사용 — Section 4.1)
- 각 단계에 `days_to_observe` 배지 ("50% = 21일 관측 필요")
- 현재 단계는 굵은 수직선 표시, 다음 단계는 점멸 마커
- 하단 Decision Gate 배지:
  - 초록: "다음 단계 확장 권장 (CI 하한도 양수)"
  - 노랑: "조건부 확장 — 관측값 검증 후"
  - 빨강: "HOLD 또는 롤백 권장"

**상호작용**:
- Fan Chart 위에서 단계 클릭 → 해당 단계의 예상 LTV, 관측 필요 일수, shrinkage 계수 툴팁 표시
- "What-if" 슬라이더: shrinkage 계수 조작 → fan 밴드 실시간 갱신 (비관/낙관 시나리오)

### 7.5 Module 5: Capital Allocation Console

**핵심 질문**: "다음에 무엇을 해야 하는가?"

| **Linear** | 우측 사이드바 동작, 키보드 우선, 미니멀 미학 | 이슈 트래커의 리스트 뷰 |
| **Bloomberg** | 색상 시맨틱 (초록=상승, 빨강=하락 절대 불변), 패널 구성 | 극단적 정보 밀도 |
| **Datadog** | 상태 바/게이지 위젯, "상태 이력" 시각화 | 인프라 모니터링의 대시보드 편집 복잡도 |
| **Amplitude** | 코호트 비교 (여러 리텐션 커브 오버레이) | "사용자가 해석해야 하는" 분석 차트 |
| **GameAnalytics** | 장르별 벤치마크 밴드 오버레이 | 위젯 기반의 복잡한 대시보드 빌더 |
| **Sensor Tower** | 업데이트 타임라인 아이콘 (차트 x축에 업데이트 시점 표시) | 전문 리서치 도구의 탐색 깊이 |
| **Carta** | 포트폴리오 뷰 (타이틀별 상태 한눈에), 시나리오 모델링 | 법률/equity 관리의 복잡한 테이블 |

## 11. 모바일 대시보드 규칙

```
화면 공간 배분 (40-30-20-10):
  40%  가장 중요한 1개 메트릭 (투자 시그널)
  30%  2-3개 보조 KPI (Payback, Burn, ROAS)
  20%  트렌드 컨텍스트 (스파크라인)
  10%  네비게이션/필터

하단 네비게이션 바:
  [Overview] [Market] [Actions] [Experiments] [Capital]
  (아이콘 + 짧은 라벨)

터치 타겟: 최소 44x44px
차트: 스파크라인/미니 게이지로 단순화 (전체 차트는 가로 모드)
```
